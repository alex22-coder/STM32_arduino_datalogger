## Data Logger на STM32F103C8T6

![даталоггер](http://aelectrik.ru/wp-content/uploads/2026/06/IMG_20260630_185517_438-1383009046.jpg)

***Описание***

Этот проект представляет собой [простой даталоггер на базе микроконтроллера STM32F103C8T6](https://ozon.ru/t/ZpFL5oP) с использованием Arduino-фреймворка. 
Устройство считывает данные с четырёх аналоговых входов и сохраняет их на SD-карту в формате CSV с временными метками (в миллисекундах от запуска).

Также на карте есть свой просмотрщик, скопируйте три файла из папки "datalogger cd" на карту, откройте файл ***index.html*** для того чтобы открыть просмотрщик

![даталоггер для автоэлектрика](http://aelectrik.ru/wp-content/uploads/2026/07/voltage.jpg)

***Основные функции***

* Логирование данных с 4 аналоговых входов (PA0–PA3)

* Сохранение данных на SD-карту в файлы формата CSV

* Автоматическое создание новых файлов с инкрементируемыми именами (log001.csv, log002.csv, ...)

* Возможность настройки интервала логирования через файл config.txt

***Светодиодная индикация состояния:***

* 3 коротких вспышки — успешная инициализация SD-карты

* Постоянное горение — выполняется запись на SD-карту

* Медленное мигание (1 Гц) — ожидание, SD-карта готова

* Быстрое мигание (2 Гц) — SD-карта не обнаружена

## Аппаратное обеспечение

* Микроконтроллер: STM32F103C8T6 (Blue Pill)

### Схема подключения

| Пин STM32 | Назначение        |
|-----------|-------------------|
| PB12      | LED (встроенный)  |
| PA4       | CS для SD-карты   |
| PB13      | SCK для SD-карты  |
| PB14      | MISO для SD-карты |
| PB15      | MOSI для SD-карты |
| PA0       | Аналоговый вход 0 |
| PA1       | Аналоговый вход 1 |
| PA2       | Аналоговый вход 2 |
| PA3       | Аналоговый вход 3 |

***Подключение к SD-модулю:***

* VCC → 3.3V или 5V (в зависимости от модуля)

* GND → GND

* CS → PA4

* SCK → PB13

* MOSI → PB15

* MISO → PB14

### Установка и настройка
1. Среда разработки
Проект написан для PlatformIO (или Arduino IDE). Для компиляции потребуется:

Установленный PlatformIO Core

Платформа ststm32

2. Сборка и прошивка
Клонируйте репозиторий и выполните:

bash
```pio run -t upload```

3. Подготовка SD-карты
Отформатируйте SD-карту в FAT32

(Опционально) создайте файл config.txt в корне карты для настройки интервала

### Настройка интервала логирования

* По умолчанию интервал логирования составляет 1000 мс (1 секунда).

* Для изменения интервала создайте на SD-карте файл config.txt со следующим содержимым:

```INTERVAL=5000```

(Где 5000 — интервал в миллисекундах).

Если файл отсутствует или не содержит корректной настройки, используется значение по умолчанию.

### Формат данных

*Данные сохраняются в файлы с именами logXXX.csv, где XXX — номер от 001 до 999. Каждый файл содержит заголовок и строки данных в формате:*

```text
Timestamp(ms),Analog_0,Analog_1,Analog_2,Analog_3
120,1024,512,256,128
1120,1020,510,260,130
...
```
```Timestamp(ms)``` — время в миллисекундах от старта микроконтроллера

```Analog_0–Analog_3``` — значения с аналоговых входов (0–4095 для STM32)

### Алгоритм работы

* Инициализация: при старте выполняется инициализация SD-карты и SPI.

* Поиск свободного номера: сканируются файлы log001.csv...log999.csv для создания нового файла с первым свободным номером.

* Создание файла: создаётся CSV-файл с заголовком.

* Чтение конфигурации: если существует config.txt, считывается интервал логирования.

**Основной цикл:**

* По таймеру (с заданным интервалом) читаются значения с 4 аналоговых входов.

* Данные записываются в открытый CSV-файл.

* Светодиод индицирует состояние.

### Важные замечания

* Используется аппаратный SPI через пины PB13–PB15.

* Аналоговые входы на STM32 имеют разрешение 12 бит (0–4095).

* При записи на SD-карту светодиод горит постоянно — это помогает визуально контролировать процесс.

* Если SD-карта не обнаружена, устройство продолжает работать, но логирование не выполняется, и светодиод быстро мигает.

### Возможные улучшения

* Добавление RTC для меток реального времени вместо millis()

* Поддержка других форматов данных (например, бинарных)

* Настройка через последовательный порт (UART)

* Использование прерываний для более точного интервала
